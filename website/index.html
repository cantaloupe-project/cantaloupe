About

Cantaloupe is an open-source image server enabling on-demand derivative generation from high-resolution source images. With available processing operations including cropping, scaling, and rotation, it can support deep-zooming image viewers, as well as on-the-fly thumbnail generation. Compliance with the IIIF Image API enables it to work with a growing number of client applications.

Developed at the University of Illinois at Urbana-Champaign Library, Cantaloupe augments an emerging digital image repository containing thousands of high-resolution scanned images. Project goals include performance, scalability, and extensibility, without sacrificing ease-of-use.

Cantaloupe features a configurable processing pipeline based on:

  • Endpoints, that intercept and translate client requests;
  • Resolvers, that locate and retrieve the source images specified in the requests;
  • Processors, that transform them according to the request parameters; and
  • Caches, that cache the generated images.

This makes it adaptable and future-ready, while remaining easy to configure and use.

Features

  • Easy to get working. Cantaloupe is an uncharacteristically simple Java application with no Servlet containers or XML files to deal with. A fast web server is built right in.
  • Self-contained for easy prototyping, deployment, and upgrading.
  • Pluggable resolvers for retrieving images from a variety of sources, with optional scriptable "hooks" to enable dynamic resolver selection and image lookup.
  • Supports opaque image identifiers for secure and stable URIs.
  • Pluggable processors to support a variety of source image formats—as well as still-frame extraction from videos in a variety of formats.
  • Optimized JPEG2000 and TIFF readers capable of excellent performance even with very large source images.
  • Optional server-side caching with configurable aggressiveness.
  • Watermarking with customizable position and minimum image size cutoff.
  • Supports fine-grained authorization policies for integration with diverse infrastructures.
