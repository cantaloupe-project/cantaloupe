What's New in 3.2

3.2

  • All processors that use the Java 2D pipeline (Java2dProcessor, KakaduProcessor, OpenJpegProcessor, PdfBoxProcessor, FfmpegProcessor) use a custom box filter for downscaling and a bicubic filter for upscaling, resulting in better performance and quality.
  • Added the ability to embed ICC color profiles in derivative images.
  • Added the ability to copy EXIF, IPTC, XMP, and/or native TIFF and PNG metadata from source images to derivative images for some processors.
  • Increased the default Java2dProcessor/JaiProcessor JPEG output quality to 0.8 (80).
  • Added support for the X-IIIF-ID proxy header, in order to return correct image identifiers in information responses whose requests have passed through a reverse proxy using a different identifier.
  • When an output format is requested that is valid but unsupported by the current processor, HTTP 415 is returned instead of HTTP 400.
